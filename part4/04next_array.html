💡 为什么是 j = next[j]？

当匹配失败时（T[i] != T[j]），我们不需要把 i 退回起点。因为此时前 j-1 个字符已经匹配成功了。

next[j] 保存的正是这前 j-1 个字符构成的子串中，最长相等前后缀的长度。让 j 退回到 next[j]，相当于直接对齐了已知的相同前后缀，极大地提升了效率！

kmp.c 

void get_next(SString T, int next[]) {
    int i = 1, j = 0;
    next[1] = 0;
    while (i < T.length) {
        if (j == 0 || T.ch[i] == T.ch[j]) {
            ++i; ++j;
            next[i] = j;
        } else {
            j = next[j];  // 回退到更短的公共前后缀
        }
    }
}
